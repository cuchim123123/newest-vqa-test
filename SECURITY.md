# Security Policy

## Supported Versions

Currently, we only provide security updates and patches for the latest major release of the Deep Learning VQA Project. 

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

Security is a top priority for our project. We take vulnerabilities in our Deep Learning models and web infrastructure seriously. If you discover a security vulnerability within this repository, **please do not disclose it publicly via standard GitHub issues.**

Instead, please report it via the following process:

1. **Direct Email:** Send a detailed report to the repository maintainers at `523H0173@student.tdtu.edu.vn` or `523H0178@student.tdtu.edu.vn`.
2. **GitHub Security Advisory:** Alternatively, you can privately report the vulnerability via the **Security** tab -> **Advisories** -> **Report a vulnerability** in this repository.

**What to expect:**
* We will acknowledge receipt of your vulnerability report within **48 hours**.
* If the vulnerability is accepted, we will work on a patch and keep you updated on the progress.
* Once the vulnerability is resolved, we will release a new version (`v1.0.x`) and publicly acknowledge your contribution (with your permission).

### Automated Security
*Note: Automated dependency updates and vulnerability scans are actively managed via **GitHub Dependabot**. Pull requests generated by Dependabot addressing dependency vulnerabilities (e.g., in `torch`, `transformers`, etc.) are reviewed and merged promptly by our team.*
